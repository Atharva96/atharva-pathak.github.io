Self-Service BI

Governed Business Access

Designed and implemented role-aware access models using Row-Level Security across Power BI and SQL layers, enabling secure, scalable self-service analytics.

RLSPower BISQL ServerData GovernanceSecurityAccess Control

1 Business Need

As the number of dashboard users grew, there was no systematic way to ensure users only saw data relevant to their role and region. Without governance, there was risk of data leakage and compliance violations.

2 What I Built

  • Multi-layered Row-Level Security at both Power BI and SQL Server levels
  • Role-to-region mapping tables for dynamic, maintainable access control
  • Dual-layer validation ensuring security at visualization and data layers
  • Admin views for access audit and role management
  • Documentation and SOPs for onboarding new users and roles

3 Tech Stack

  • Power BI: DAX-based RLS roles with dynamic security filters
  • SQL Server: Server-side RLS using security predicates
  • Governance: Access control matrices and audit logging
  • Documentation: SOPs for role management and onboarding
  • Testing: UAT validation across all role permutations

4 Impact & Outcomes

  • Enabled secure self-service analytics for 100+ users across departments
  • Zero data leakage incidents since implementation
  • Improved analytics adoption — users trust the data they see
  • Reduced admin overhead through automated role-mapping
  • Compliance-ready access model supporting audit requirements
